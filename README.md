# Async Data Job API

大規模 JSON / NDJSON データを非同期で処理するジョブ基盤 API。

## Features

- **ジョブ管理**: 作成・一覧・詳細・キャンセル API
- **状態管理**: `pending` → `running` → `succeeded` / `failed` / `canceled`
- **非同期ワーカー**: goroutine pool による並行処理（同時実行数制限付き）
- **バックグラウンドポーリング**: 高負荷時や再起動時に未処理の滞留ジョブ（Pending状態）を自動復旧
- **JSON / NDJSON ストリーム処理**: 行単位読み込みとメモリ最適化で大容量ファイルへ安定対応
- **リトライ**: 設定可能な最大リトライ回数、失敗時は自動再キュー
- **DLQ (Dead Letter Queue)**: 失敗履歴をテーブルに記録
- **デタラメではない進捗**: DBへのUPDATE頻度をスロットリングし競合を抑制
- **冪等性キー**: 同一キーによる重複作成防止
- **入力バリデーション**: リクエスト検証
- **タイムアウト**: ジョブ単位のタイムアウト制御
- **構造化ログ**: `log/slog` JSON ハンドラ
- **メトリクス**: Prometheus (Counter / Histogram)
- **分散トレーシング**: OpenTelemetry (環境変数でJaeger等への出力先を動的に変更可能)
- **OpenAPI Spec**: `docs/openapi.yml`

## Quick Start

```bash
# 起動（PostgreSQL + migrate + app + Prometheus）
make up

# ヘルスチェック
curl http://localhost:8080/healthz

# ジョブ作成
curl -X POST http://localhost:8080/api/v1/jobs \
  -H "Content-Type: application/json" \
  -d '{"input_url": "https://jsonplaceholder.typicode.com/posts"}'

# シードデータ投入
make up-seed
```

## API Endpoints

| Method | Path | Description |
|---|---|---|
| `POST` | `/api/v1/jobs` | ジョブ作成 |
| `GET` | `/api/v1/jobs` | ジョブ一覧 (pagination, status filter) |
| `GET` | `/api/v1/jobs/{id}` | ジョブ詳細 |
| `POST` | `/api/v1/jobs/{id}/cancel` | ジョブキャンセル |
| `GET` | `/api/v1/jobs/{id}/failures` | DLQ 失敗履歴 |
| `GET` | `/healthz` | ヘルスチェック |
| `GET` | `/metrics` | Prometheus メトリクス |

## Configuration

| Env Variable | Default | Description |
|---|---|---|
| `DATABASE_URL` | `postgres://jobapi:jobapi@localhost:5432/jobapi?sslmode=disable` | PostgreSQL 接続文字列 |
| `SERVER_PORT` | `8080` | HTTP ポート |
| `WORKER_CONCURRENCY` | `4` | ワーカー同時実行数 |
| `WORKER_TIMEOUT_SEC` | `300` | ジョブタイムアウト（秒） |
| `WORKER_MAX_RETRIES` | `3` | 最大リトライ回数 |
| `LOG_LEVEL` | `info` | ログレベル (debug/info/warn/error) |
| `OTEL_EXPORTER_OTLP_ENDPOINT` | (空) | 空の場合は標準出力にトレースする。`http://jaeger:4318` 等を指定すると OTLP 経由でダッシュボードへ送信される |
